## Angular CLI

The **Angular CLI** is a command-line interface tool that you use to initialize, develop, scaffold, and maintain Angular applications.  

You already used the CLI to serve the application locally with the `ng serve` command in the <a href="/practical-course/getting-started/#running-the-application" target="_blank">Getting Started</a> step.

Now it's time use it to create a new component.

#### üéì Instructions

1. Open another terminal tab (<a href="https://code.visualstudio.com/docs/terminal/basics" target="_blank">VSCode docs</a>).
2. Run the following command:

```bash
ng generate component task-list
```

3. You should see a new folder called `task-list` in the `src/app` folder.


:::note
This new folder contains the three following files:

- `task-list.component.ts`
- `task-list.component.html`
- `task-list.component.css`
:::

## Component metadata

By opening the `task-list.component.ts` file, you can see the following code:

```typescript
// task-list.component.ts
@Component({
  selector: 'app-task-list',
  templateUrl: './task-list.component.html',
  styleUrls: ['./task-list.component.css']
})
export class TaskListComponent implements OnInit {

  ngOnInit() {}
}
```

The `@Component` decorator define the metadata for the component.

The `selector` property defines the component's CSS element selector.
By using this selector in a template HTML file, Angular creates and displays an instance of the component.

The `templateUrl` property defines the HTML template file for the component.
That's the template that will be displayed when the css selector is used in another template.

The `styleUrls` property defines the CSS files for the component.
These styles will be applied to the component template.


## Display the component

So far the application only displays the `app` component html content.
As the `task-list` component selector is not used in the `app` component, it's not displayed in the browser.

#### üéì Instructions

1. Open the `src/app/app.component.html` file.
2. Add the task list selector as followed:

```html ins={"Add the task list selector and the main tag for styling and semantic purposes": 5-8}
// app.component.html
<header>
    <h1>Angular Practical course</h1>
</header>

<main class="container pt-4">
  <app-task-list />
</main>
```

3. Check out the changes in your browser.

:::tip
`task-list works!` is displayed in the browser.
That's the default content generated by the Angular CLI in the component html file.
:::
